# Ollama Planner Proof

TaskFlow supports optional local LLM workflow planning through Ollama.

## Model Used

llama3.2:1b

## Successful Planner Metadata

When Ollama is running and the model returns valid structured JSON, the API response includes:

{
  "planner": {
    "mode": "ollama",
    "model": "llama3.2:1b",
    "llm_available": true
  }
}

## Meaning

This confirms that the workflow plan was generated by the local Ollama model rather than the rule-based fallback.

## Fallback Behaviour

If Ollama is unavailable, TaskFlow safely falls back to the rule-based planner and returns metadata like:

{
  "planner": {
    "mode": "rule_based_fallback_ollama_unavailable",
    "model": "llama3.2:1b",
    "llm_available": false
  }
}

This makes the system transparent and reliable even when the local model is not running.
